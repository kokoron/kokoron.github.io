初めてのVapor

サーバーサイドSwiftを試しにやってみようかと思い立ち、試しにVaporを触ってみた話

April 2, 2017 Kokoro

1 minute read

  • Twitter

  • Google+

  • Facebook

  • Reddit

  • LinkedIn

  • StumbleUpon

  • Email

先日 try! Swift に行ってきたんですが、話題として結構サーバーサイドSwiftの話も出ていて、サーバー弱者な僕としても少しくらいはサーバーサイドのこともわかっておかないとなあ・・・と思いサーバーサイドSwiftを試しにやってみようかと思い立ち、試しにVaporを触ってみた話です。

とりあえずインストールからherokuにデプロイするまでをやってみました。

まずはVaporのインストール

まずはXCode8をインストールします。ていうかもう入ってます。

ターミナルで以下のコマンドで環境をチェック。

$ curl -sL check.vapor.sh | bash

Toolboxをインストール

$ curl -sL toolbox.vapor.sh | bash

App作成

新規Appを作成します。

$ vapor new TestBot

$ cd TestBot
$ vapor build
$ vapor run

ディレクトリ内に移動してbuildしてrunします。

Running RedishBot...

GET /
GET /styles/app.css

http://localhost:8080 をブラウザで開くと

おお！動いてる！

herokuにDeploy

上記Appのリポジトリを作り

herokuにログイン

$ heroku login

Appを作成してbuildpackを設定してherokuにmasterをpushします。

$ heroku create
$ heroku buildpacks:set https://github.com/kylef/heroku-buildpack-swift
$ git push heroku master

しばらく待つと

remote: Verifying deploy... done.

となるので、https://xxxxxxxxxx.herokuapp.com/（さっき作ったherokuのアドレス）をブラウザで開くと・・・

おお！(；´Д｀) 動いてる！！
